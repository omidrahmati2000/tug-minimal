Entity Relationships

System Admin Management: Direct role assignment via system_roles table
Organization Members: Role assignment via organization_users table
API Key Management: Centralized in Authentication Service
Station Authentication: API keys referenced from Auth Service
Card Assignment: Cards linked to users within organizations
Transaction Processing: Links cards, organizations, and stations
System Administrator Management
System administrators are managed through the system_roles table, providing direct role assignment without requiring organization membership. This allows platform-level administration with full system access.
API Key Centralization
All API keys are managed in the Authentication Service database (api_keys table) for centralized security, rate limiting, and access control. Other services reference these keys via foreign key relationships.
Permission Resolution Logic
IF user IN system_roles THEN check_system_permissions()
ELSE IF user IN organization_users THEN check_org_permissions()
ELSE deny_access()
